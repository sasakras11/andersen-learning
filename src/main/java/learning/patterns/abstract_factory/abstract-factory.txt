Якщо нам необхідно віддавати клієнту набір об'єктів різного сімейства але
які між собою зв'язані - нам потрібно використовувати паттерн Абстрактна фабрика

Як його реалізувати

1.) Визначити типи, які будуть повторюватися між симействами і створити для них інтерфейси.
2.) Створити інтерфейс, абстрактний клас який буде нам створювати ці об'єкти ( фабрика)
3.) Для кожного сімейства описати Конкретні реалізації і написати реалізацію конкретнох фабрики для сімейства

Приклад

1.) Ми даємо клієнту дектопні програми для роботи з фото, фідео та текстом ( cтворюємо інтерфейси PhotoEditor,
TextEditor,VideoMaker)
2.) ствопюємо інтерфейс в якому описані методи що будуть вертати об'єкти
3.) Створюємо реалізації програм та фабрику, яка описує систему - ( Microsoft Word, Photoshop,Vimeo Studio - WINDOWS),
( Flow Blade,GIMP,Sublime - LINUX), (TextEdit,IMovie,Luminar - Mac)

Переваги :

1.) клієнтський код не залежний від конкретних реалізацій
2.) реалізує принцип open/close

Недоліки :

1.) Усложняє код бо добавляється багато нових класів
